## Product Overview

This platform connects mentors and mentees to learn skills through booked sessions (Google Meet calls).
Users can act as both mentors and mentees.

The system is built around:

* skill-based discovery (no name search)
* session-based interactions
* rating-driven credit rewards
* trust and anti-fraud mechanisms

---

## Core Rules

### 1. Discovery

* Users can ONLY search by:

  * skills
  * keywords
* Users CANNOT search by:

  * name
  * username
* AI (Claude) maps queries to relevant mentors.

---

### 2. Sessions

* All interactions must occur through booked sessions.
* Sessions are created via:

  * Google Calendar
  * Google Meet link
* Session must be validated using:
  * Manual confirmation from participants (Honor System)

---

### 3. Ratings

* Only mentees rate mentors.
* Each session can be rated **once only**.
* Ratings are optional (no blocking).
* Ratings must be tied to a valid session.

---

### 4. Credits System

* Users start with 3 credits.
* Credits are **NOT spent or deducted**.
* Credits are ONLY earned from high ratings.
* Credits act as a **reputation signal**, not currency.

---

### 5. Mentor Ranking

Mentors are ranked based on:

* skill relevance
* rating score
* number of sessions
* verification status
* early boost for new mentors

---

### 6. User Roles

* Every user can be BOTH:

  * mentor
  * mentee

---

### 7. Profile

Users can add:

* hashtags (skills)
* school
* degree
* bio

---

### 8. Verification

* Google OAuth required
* No email/password auth in the app
* School/work email optional (boosts trust)

---

### 9. Trust & Safety

User states:

* Active
* Flagged
* Warning issued
* Temporary ban
* Tribunal review
* Permanent ban

Triggers:

* repeated low ratings
* disputes
* suspicious patterns

---

### 10. Fraud Prevention

* No name search (prevents friend farming)
* Ratings only from real sessions
* One rating per session
* Repeat interactions may be down-weighted
* Session validation required (Manual confirmation)

---

### 11. History

Users can view:

* past sessions
* mentors/mentees interacted with
* ratings
* credits earned

---

### 12. Disputes & Refunds

* Users can file disputes after a session
* Disputes pause credit rewards
* Admin/tribunal resolves outcome
* Refund logic applies if needed

---

## Key Philosophy

* Ratings = trust signal
* Credits = reputation reward
* Sessions = only valid interaction
* System must resist abuse by design
